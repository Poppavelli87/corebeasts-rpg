# Security Policy

## Reporting a Vulnerability

Please do not open a public issue for security vulnerabilities.

Instead, report privately by contacting the maintainers directly (or via GitHub private vulnerability reporting if enabled for the repository).

Include:

- A clear description of the issue
- Steps to reproduce
- Potential impact
- Suggested mitigation (if known)

We will acknowledge reports and work on a fix as quickly as possible.

## Supported Versions

This project is under active development. Security fixes are applied to the latest `main` branch and latest release tag.
